# show-violations

Show policy violations from Kyverno PolicyReport and ClusterPolicyReport resources. Returns all non-passing results (fail, error, warn).

## When to Use

✅ **USE this skill when:**
- Viewing current policy violations in the cluster
- Generating compliance reports
- Monitoring policy enforcement after deployment
- Troubleshooting why resources are being blocked
- Checking which resources violate specific policies
- Daily/weekly compliance checks

## When NOT to Use

❌ **DON'T use this skill when:**
- Dry-run validation before deployment (use kyverno-cli-audit instead)
- Deploying policies (use deploy-policies instead)
- Checking if Kyverno is installed (use check-kyverno instead)
- Installing Kyverno (use install-kyverno instead)
- Getting help with Kyverno (use kyverno-help instead)
- Kyverno is not installed yet (will fail)

## Commands

### Query commands

```bash
# Get PolicyReports (namespaced)
kubectl get policyreports -n default

# Get ClusterPolicyReports (cluster-wide)
kubectl get clusterpolicyreports

# Get detailed report
kubectl get policyreport -n default -o yaml

# Get specific violation details
kubectl get policyreport -n default -o json | jq '.results[] | select(.result == "fail")'
```

### Filter commands

```bash
# Filter by severity
kubectl get clusterpolicyreports -o json | jq '.items[].results[] | select(.severity == "high")'

# Filter by policy
kubectl get policyreports -n default -o json | jq '.results[] | select(.policy == "require-run-as-nonroot")'
```

## Parameters

- **namespace** - Namespace to query (default: `default`, use `all` for all namespaces)
- **namespace_exclude** - Namespaces to exclude (default: `kube-system,kyverno`)

## Output Format

Returns JSON array of violations:

```json
[
  {
    "policy": "require-run-as-nonroot",
    "rule": "run-as-non-root",
    "message": "Validation rule 'run-as-non-root' failed",
    "category": "Pod Security",
    "severity": "high",
    "result": "fail",
    "resources": [
      "Pod/default/nginx-pod"
    ],
    "timestamp": "2024-01-15T10:30:00Z"
  }
]
```

## Fields

- **policy** - Policy name
- **rule** - Rule name
- **message** - Violation message
- **category** - Policy category
- **severity** - Severity level (critical, high, medium, low)
- **result** - Result status (fail, error, warn)
- **resources** - List of affected resources
- **timestamp** - When violation was detected

## Result Types

### fail
Resource violates policy and action was blocked (Enforce mode).

### error
Error occurred during policy evaluation.

### warn
Resource violates policy but only logged (Audit mode).

## Common Workflows

### Daily compliance check

```
# Check all violations
kyverno violations --namespace all

# Generate report
kyverno violations --namespace all > compliance-report.json

# Fix critical and high severity issues
```

### Namespace-specific audit

```
# Check production namespace
kyverno violations --namespace production

# Check development namespace
kyverno violations --namespace development
```

### Exclude system namespaces

```
# Exclude additional namespaces
kyverno violations --namespace all --namespace_exclude "kube-system,kyverno,monitoring"
```

### After policy deployment

```
# 1. Deploy policies
kyverno deploy --policySets pod-security

# 2. Wait for background scan
sleep 60

# 3. Check violations
kyverno violations --namespace all
```

## Notes

- Requires Kyverno to be installed with PolicyReport CRDs
- Only returns fail, error, and warn results (pass results filtered out)
- Automatically excludes specified namespaces when querying all
- If Kyverno not installed, returns installation instructions
- Use with check-kyverno to verify installation first
- Reports generated by background controller (may have delay)
- ClusterPolicyReports contain cluster-wide violations
- PolicyReports are namespace-scoped
